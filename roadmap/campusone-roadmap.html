Technical Blueprint
App Folder Structure (Feature-First approach)
lib/
├── screens/// Core App Screen UIs
├── widgets/// Global Reusable Custom UI Elements
├── models/// Dart Data classes for Firestore serialization
├── services/// Firebase Auth, Firestore streams singletons
├── providers/// State management (Provider/Riverpod Controllers)
├── utils/// Helpers, formatters, inputs validation rules
├── themes/// Light/Dark style variables (Poppins config)
└── main.dart// Application initialization gateway

Architecture Highlights

Clean, scalable, aur decouple approach follow kiya gaya hai. State management aur network/database protocols rules ko views layer se bilkul separate rakha hai taaki feature scaling me asani ho.
